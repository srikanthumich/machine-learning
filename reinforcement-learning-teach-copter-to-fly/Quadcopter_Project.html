Reinforcement Learning¶

Project: Train a Quadcopter How to Fly¶

In this project we'll design an agent to fly a quadcopter, and then train it using a reinforcement learning algorithm!

Structure¶

Take a look at the files in the directory to better understand the structure of the project.

  • task.py: Defines the task (environment).
  • agents/: Folder containing reinforcement learning agents.
    • policy_search.py: A sample agent is provided here.
    • agent.py: Develop our agent.
  • physics_sim.py: Contains the simulator for the quadcopter.

For this project, we will define our own task in task.py. We will also design a reinforcement learning agent in agent.py to complete the chosen task. We will also define a model.py file defining needed neural network architectures.

Controlling the Quadcopter¶

A sample agent is given in the code cell below to show how to use the sim to control the quadcopter. This agent is even simpler than the sample agent that we'll examine (in agents/policy_search.py) later in this notebook!

The agent controls the quadcopter by setting the revolutions per second on each of its four rotors. The provided agent in the Basic_Agent class below always selects a random action for each of the four rotors. These four speeds are returned by the act method as a list of four floating-point numbers.

For this project, the agent that we will implement in agents/agent.py will have a far more intelligent method for selecting actions!

import random

class Basic_Agent():
    def __init__(self, task):
        self.task = task
    
    def act(self):
        new_thrust = random.gauss(450., 25.)
        return [new_thrust + random.gauss(0., 1.) for x in range(4)]

        _, _, done = task.step(rotor_speeds)
        to_write = [task.sim.time] + list(task.sim.pose) + list(task.sim.v) + list(task.sim.angular_v) + list(rotor_speeds)
        for ii in range(len(labels)):
            results[labels[ii]].append(to_write[ii])
        writer.writerow(to_write)
        if done:
            break

In the sample task in task.py, a 6-dimensional pose of the quadcopter is used to construct the state of the environment at each timestep. However, when amending the task for our purposes, we can expand the size of the state vector by including the velocity information. We can use any combination of the pose, velocity, and angular velocity to construct the state to suit our task.

The Task¶

A sample task is shown in task.py.

The __init__() method is used to initialize several variables that are needed to specify the task.

  • The simulator is initialized as an instance of the PhysicsSim class (from physics_sim.py).
  • Inspired by the methodology in the original DDPG paper, action repeats are used. For each timestep of the agent, we step the simulation action_repeats timesteps. More can be read about action repeats in the Results section of the DDPG paper.
  • We set the number of elements in the state vector. For the sample task, we only work with the 6-dimensional pose information. To set the size of the state (state_size), we must take action repeats into account.
  • The environment will always have a 4-dimensional action space, with one entry for each rotor (action_size=4). We can set the minimum (action_low) and maximum (action_high) values of each entry here.
  • The sample task in this provided file is for the agent to reach a target position. We specify that target position as a variable.

The reset() method resets the simulator. The agent should call this method every time the episode ends. We can see an example of this in the code cell below.

The step() method is perhaps the most important. It accepts the agent's choice of action rotor_speeds, which is used to prepare the next state to pass on to the agent. Then, the reward is computed from get_reward(). The episode is considered done if the time limit has been exceeded, or the quadcopter has travelled outside of the bounds of the simulation.

In the next section, we will test the performance of an agent on this task.

The Agent¶

In the sample agent in agents/policy_search.py we use a very simplistic linear policy to directly compute the action vector as a dot product of the state vector and a matrix of weights. Then, it randomly perturbs the parameters by adding some Gaussian noise, to produce a different policy. Based on the average reward obtained in each episode (score), it keeps track of the best set of parameters found so far, how the score is changing, and accordingly tweaks a scaling factor to widen or tighten the noise.

Define the Task, Design the Agent, and Train the Agent!¶

We'll amend task.py to specify a task of our choosing. The common tasks to specify could be: teaching our quadcopter to takeoff, hovering in place, landing softly, or reaching a target pose. After specifying a task, we use the sample agent in agents/policy_search.py as a template to define our own agent in agents/agent.py.

Note that it is highly unlikely that the first agent and task that we specify will learn well. We will likely have to tweak various hyperparameters and the reward function for our task until we arrive at reasonably good behavior.

As we develop our agent, it's important to keep an eye on how it's performing. We'll build in a mechanism to log/save the total rewards obtained in each episode to file. If the episode rewards are gradually increasing, this is an indication that our agent is learning.

## Train your agent here.
import sys
import csv
import numpy as np
import pandas as pd
from task import Task
from agents.agent import DDPG

# Take-off task 

# Initial pose for landing 
runtime = 5. 
init_pose = np.array([0., 0., 0., 0., 0., 0.])
init_velocities = np.array([0., 0., 0.])
init_angle_velocities = np.array([0., 0., 0.])
file_output = 'rewards.txt'
episodes = 1000
target_pos = np.array([0., 0., 10.])
done = False

# Setup in case of DDPG agent 
task = Task(target_pos = target_pos)
agent = DDPG(task)
labels = ['episode', 'rewards_total']
results = {x : [] for x in labels}

# Running sim and saving results.
with open(file_output, 'w') as csvfile:
    writer = csv.writer(csvfile)
    writer.writerow(labels)
    best_total_reward = 0
    for i in range(1, episodes+1):
        state = agent.reset_episode()
        total_reward = 0
        while True:
            action = agent.act(state)
            next_state, reward, done = task.step(action)
            total_reward += reward
            if total_reward > best_total_reward:
                best_total_reward = total_reward
            agent.step(action, reward, next_state, done)
            state = next_state
            if done:
                to_write = [i] + [total_reward]
                for ii in range(len(labels)):
                    results[labels[ii]].append(to_write[ii])
                writer.writerow(to_write)
                print("\rEpisode = {:4d}, total_reward = {:7.3f} (best = {:7.3f})".format(
                    i, total_reward, best_total_reward), end="")
                break
        sys.stdout.flush()

Episode = 1000, total_reward =  20.253 (best =  53.858))
